Building template partials

A partial is a template snippet which can be inserted into templates, or even other partials. They help to keep templates uncluttered and easy to read.

Are you going to be using any forms on your site that appear in different areas or templates? Create a partial template for the markup. For a newsletter sign up form, you would start off by adding a new file to the resources/views/partials/ directory:

<!-- @ resources/views/partials/form-newsletter.blade.php -->
<form action="https://example.com/subscribe" method="post">
  <input type="email" value="" name="email" placeholder="Your email address">
  <button type="submit">Sign me up!</button>
</form>

Now you can include your newsletter partial in other templates:

@include('partials.form-newsletter')

Other examples of partials you can create:

  • Social network links

    <!-- @ resources/views/partials/list-social.blade.php -->

<ul class="list-social list-inline">
  <li class="list-inline-item"><a href="{{ get_field('social_twitter', 'option')) }}"><i class="fa fa-twitter-square" aria-hidden="true"></i> <span class="sr-only">Twitter</span></a></li>
  <li class="list-inline-item"><a href="{{ get_field('social_facebook', 'option')) }}"><i class="fa fa-facebook-square" aria-hidden="true"></i> <span class="sr-only">Facebook</span></a></li>
</ul>

  • Author biographies (use on single post template as well as author archive templates)

    <!-- @ resources/views/partials/entry-author-info.blade.php -->

<div class="entry-author-info">
  <h4>About the author</h4>
  <p>
    <img src="https://www.gravatar.com/avatar/{{ md5(get_the_author_meta('email')) }}?s=180" width="90" height="90" class="img-circle alignleft" alt="">
    {{ get_the_author_meta('description') }}
  </p>
</div>

  • Modals

    <!-- @ resources/views/partials/modal-newsletter.blade.php -->
<div class="modal fade" id="modal-newsletter" tabindex="-1" role="dialog" aria-hidden="true">
  <div class="modal-dialog" role="document">
    <div class="modal-content">
      ...
    </div>
  </div>
</div>

  • Call to actions

    <!-- @ resources/views/partials/tour-cta.blade.php -->
<section class="tour-cta">
  <div class="container">
    <div class="row">
      <div class="col-sm-8 offset-sm-2">
        <h3>Set Up Your Tour</h3>
        @php gravity_form(1, false, false) @endphp
      </div>
    </div>
  </div>
</section>

If you're ever in a situation where you'd be using the same component across multiple templates, create a partial for it.

Passing data to partials

You can pass an array of data when calling @include to pass data to use in a view. The example below passes img_id and img_url variables to be used in the partials/content-single.blade.php view:

<!-- @ resources/views/single.blade.php -->
@while(have_posts()) @php(the_post())
  @include('partials/content-single-'.get_post_type(), ['img_id' => get_post_thumnail_id(), 'img_url' => wp_get_attachment_image_src(get_post_thumnail_id(), 'large', true)])
@endwhile

<!-- @ resources/views/partials/content-single.blade.php -->
<header style="background-image: url({{ $img_url[0] }});">
