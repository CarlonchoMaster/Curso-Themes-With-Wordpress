Template files

There’s nothing different about the normal WordPress template hierarchy in Sage. Until you get the hang of the naming conventions, keep a template hierarchy reference close by.

resources/views/
├── 404.blade.php
├── index.blade.php
├── layouts/
│   └── app.blade.php
├── page.blade.php
├── partials/
│   ├── comments.blade.php
│   ├── content-page.blade.php
│   ├── content-search.blade.php
│   ├── content-single.blade.php
│   ├── content.blade.php
│   ├── entry-meta.blade.php
│   ├── footer.blade.php
│   ├── head.blade.php
│   ├── header.blade.php
│   ├── page-header.blade.php
│   └── sidebar.blade.php
├── search.blade.php
├── single.blade.php
└── template-custom.blade.php

The resources/views/ directory is where you place new template files, such as custom archive templates.

The resources/views/layouts/ directory is where your base templates live. resources/views/layouts/app.blade.php is the base template file that wraps each template.

The resources/views/partials/ directory is where you place template partials, or re-usable template parts, that are called from other templates.

Page titles

At the top of most page templates, the page header is included with:

@include('partials.page-header')

resources/views/partials/page-header.blade.php references the title function found in app/helpers.php:

function title()
{
    if (is_home()) {
        if ($home = get_option('page_for_posts', true)) {
            return get_the_title($home);
        }
        return __('Latest Posts', 'sage');
    }
    if (is_archive()) {
        return get_the_archive_title();
    }
    if (is_search()) {
        return sprintf(__('Search Results for %s', 'sage'), get_search_query());
    }
    if (is_404()) {
        return __('Not Found', 'sage');
    }
    return get_the_title();
}

The page title for a blog page comes from the title of that page itself.

Archive templates (authors, categories, post types, taxonomies, tags) titles come from the get_the_archive_title WordPress core function. To customize archive template titles, add the following to app/filters.php:

add_filter('get_the_archive_title', function ($title) {
    // Get title for Team post type archive from ACF option
    if (get_post_type('team')) {
        $title = get_field('team_title', 'option');
    }

    return $title;
});

Search results and 404 titles are also handled at the end of the title function. We encourage you to edit the function directly if you're wanting to change the functionality, but you can also:

  • Edit resources/views/partials/page-header.blade.php and add conditionals within there, eg.:
    <div class="page-header">
  @if (is_home())
    <h1>Blog Home Title</h1>
  @else
    <h1>{!! App\title() !!}</h1>
  @endif
</div>
  • Edit the page template directly, removing @include('partials.page-header') and adding your title however you want:
    <h1>Page not found!</h1>
<div class="alert alert-warning">
  {{ __('Sorry, but the page you were trying to view does not exist.', 'sage') }}
</div>

Home page template

Sage doesn’t include a specific page template for a static front page. To setup a home page template, the first thing you need to do is copy resources/views/page.blade.php to resources/views/front-page.blade.php. During your initial template development it’s okay to put hard-coded content in your template files. Further down the line while developing your theme, you’ll want to revisit your hardcoded template files and update them to be managed by custom fields or with the theme customizer.

If you're using a static front page instead of a blog, make sure the reading settings in the WordPress admin reflect that.

Blog template

Sage doesn’t include a specific page template for the main blog page. The first thing you need to do is copy resources/views/index.blade.php, the default archive template, to resources/views/home.blade.php.

Custom post type archive templates, category templates

resources/views/index.blade.php is the fallback for all of the archive templates (authors, categories, post types, taxonomies, tags). The main piece of these templates is where it grabs the content from:

@while (have_posts()) @php the_post() @endphp
  @include ('partials.content')
@endwhile

This allows you to extend partials/content.blade.php in a few different ways. content.blade.php is the partial that's used to show a post. Out of the box it looks like:

<article @php(post_class())>
  <header>
    <h2 class="entry-title"><a href="{{ get_permalink() }}">{{ get_the_title() }}</a></h2>
    @include('partials/entry-meta')
  </header>
  <div class="entry-summary">
    @php(the_excerpt())
  </div>
</article>

If your theme uses a custom post type named team, the following template file would attempt to be loaded:

resources/views/partials/content-team.blade.php

The contents of content-team.blade.php don't need to resemble content.blade.php at all. Here's a customized example that includes ACF fields:

<article @php(post_class('team-member row'))>
  <div class="col-md-4">
    <p><a href="{{ get_permalink() }}">{!! the_post_thumbnail() !!}</a></p>
  </div>
  <div class="col-md-8 text-md-left">
    <h2 class="entry-title">
      {{ get_the_title() }}
      <small>{{ get_field('member_title') }}</small>
    </h2>
    <div class="entry-summary hidden-sm-down">
      @php(the_excerpt())
    </div>
  </div>
</article>

To create a custom page template for the team archive page, copy index.blade.php to archive-<post_type>.blade.php:

# @ themes/your-theme-name/
$ cp resources/views/index.blade.php resources/views/archive-team.blade.php

Page templates

Use template-custom.blade.php as a guide to create your own custom page templates. Custom page templates are similar to a normal page.blade.php but with the template name defined in a PHP comment.

Avoid creating templates with names that are based on the slug, such as page-<slug>.blade.php. Page templates based on the slug automatically load, don’t require the template name in a PHP comment, and don’t require you to manually select the page template. That might be convenient at first, but the template would stop working if the page slug were to be changed. Instead, create templates based on the template-<name>.blade.php format.

To create a custom page template for the Contact page:

# @ themes/your-theme-name/
$ cp resources/views/template-custom.blade.php resources/views/template-contact.blade.php

Then open up template-contact.blade.php and update the comment at the top of the file to reflect the template name:

{{--
  Template Name: Contact Us
--}}

Single post, custom post type templates

resources/views/single.blade.php calls the single partial:

@include('partials/content-single-'.get_post_type())

This allows you to extend partials/content-single.blade.php for your custom post types. For the team post type, the following template file would attempt to load:

@include('partials/content-single')

Other templates

Copy existing templates to new files when necessary to fit your needs. Follow the WordPress template hierarchy to find out the appropriate file names to use, making sure to use .blade.php instead of just .php for the file extension.

Don't be shy about creating new templates, and don't feel the need to be constrained to what's in Sage out of the box.
