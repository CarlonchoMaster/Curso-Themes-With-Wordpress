Displaying the sidebar

Previous versions of Sage shipped with the primary sidebar enabled by default on all pages except for the front page and the 404 page.

Sage 9 does not come with the primary sidebar enabled on any pages.

In resources/views/layouts/app.blade.php you'll find:

@if (App\display_sidebar())
  <aside class="sidebar">
    @include('partials.sidebar')
  </aside>
@endif

Sage includes a sage/display_sidebar filter that can be used to define which conditions to enable the sidebar on.

The following snippet can be added to app/filters.php to enable the sidebar on single posts:

add_filter('sage/display_sidebar', function () {
    if (is_single()) {
        return true;
    }
});

An example of using multiple conditionals to enable sidebar:

add_filter('sage/display_sidebar', function ($display) {
    static $display;

    isset($display) || $display = in_array(true, [
      // The sidebar will be displayed if any of the following return true
      is_single(),
      is_404(),
      is_search()
    ]);

    return $display;
});

You can also manually include sidebars in your template files the same way the primary sidebar is included in resources/views/partials/sidebar.blade.php:

@php(dynamic_sidebar('sidebar-primary'))

If you want to change which sidebar is used in the primary sidebar area, add conditionals to resources/views/partials/sidebar.blade.php:

@if (is_blog())
  @php(dynamic_sidebar('sidebar-blog'))
@else
  @php(dynamic_sidebar('sidebar-primary'))
@endif
