Theme setup

Sage uses app/setup.php to enqueue front-end assets and enable theme features.

The app/setup.php file is also used to register navigation menus, sidebars, and define theme support for WordPress core functionality.

Soil features

If you install and activate our Soil plugin, Sage will automatically enable additional features including:

  • Load jQuery from the jQuery CDN
    jQuery is loaded via a CDN rather than your webserver to benefit from speed and caching
  • Cleaner WordPress markup
    The front-end output WordPress is pretty dirty. There's unnecessary scripts, styles, <meta> and <link>'s.
  • Cleaner walker for navigation menus
    Removes id and uses a simple class="menu-<title>" format for each menu item
  • Nice search
    Redirects /?s=query to /search/query

  • Root relative URLs
    Most, but not all, URLs from WordPress are changed to be root relative:

    <!-- Without Soil -->
<link rel="stylesheet" href="http://example.com/wp/wp-content/themes/twentysixteen/style.css">

<!-- With Soil -->
<link rel="stylesheet" href="/wp/wp-content/themes/twentysixteen/style.css">

While Soil is optional, we heavily recommend it primarily to have cleaner output of WordPress markup. WordPress isn't particularly known for having a nice codebase, and that is evident in both the core PHP code as well as front-end code that ends up on your site from core.

WordPress sites without Soil lead to large page sizes and longer load times. Remember that Google punishes you for bad performance (since 2010 it has used site speed in search ranking). As a developer you should strive for optimal performance, and Soil takes some of that work out of your hands.

Navigation menus

If you plan on having additional navigation menus that aren’t the same as your primary navigation, add them in the register_nav_menus array. Here we’ve added utility navigation and footer navigation menus to the array:

register_nav_menus([
  'primary_navigation' => __('Primary Navigation', 'sage'),
  'utility_navigation' => __('Utility Navigation', 'sage'),
  'footer_navigation' => __('Footer Navigation', 'sage')
]);

After your new navigation menu locations are registered, you can create new menus in the WordPress admin and then assign them to the appropriate location. Better yet, you could use WP-CLI to do it:

$ wp menu create "Footer Navigation"
Success: Created menu 3.
$ wp menu location assign 3 footer_navigation
Success: Assigned location to menu.

Navigation menus can be included in your template files the same way the primary navigation is included in templates/partials/header.php:

<?php wp_nav_menu(['theme_location' => 'footer_navigation', 'menu_class' => 'nav']); ?>

Custom image sizes

In app/setup.php you’ll find this line:

add_theme_support('post-thumbnails');

Below this line is where you’re going to want to define your custom image sizes. Whenever an image is uploaded through the WordPress admin, additional copies of the image are saved based on the registered image sizes.

add_theme_support('post-thumbnails');
add_image_size('post-featured', 1280, 800, true);

The default WordPress image names and sizes are:

the_post_thumbnail('thumbnail');    // 150x150, hard cropped
the_post_thumbnail('medium');       // 300x300, 300px max height
the_post_thumbnail('medium_large'); // 768px wide
the_post_thumbnail('large');        // 1024x1024, 1024px max height
the_post_thumbnail('full');         // Original

In the next chapter we'll go over including post thumbnails in your template files.

Defining sidebars

Sage includes two sidebar areas by default: primary sidebar & footer. The registered sidebars are shown in the WordPress admin where widgets are customized — both in the theme customizer as well as the widgets page.

Whenever you want an area to be managed by WordPress widgets, create additional sidebars. Here we’re adding a new sidebar for the Blog page by copying the register code from one of the existing sidebars:

register_sidebar([
    'name'          => __('Blog', 'sage'),
    'id'            => 'sidebar-blog'
] + $config);
