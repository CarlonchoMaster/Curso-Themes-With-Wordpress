Customizing the front-end

Things to keep in mind about a a default Sage installation:

  • The HTML (mostly found in templates/layouts/base.php) is originally based on HTML5 Boilerplate, with ARIA roles and other microformats used throughout the templates (HTML5 is the current version of the HTML standard)
  • Bootstrap 4 is included as the default option (via sage-installer):
Which framework would you like to load? [Bootstrap]:
 [0] None
 [1] Bootstrap
 [2] Bulma
 [3] Foundation
 [4] Tachyons
  • Want to remove Bootstrap and replace it with another framework, or just have a completely fresh start?
  • Want to use xyz, or just have a completely fresh start?

  • Memorize the CSS setup and organization:

    assets/styles/
├── common/      # Variables, global styles
├── components/  # Theme components (forms, buttons, etc.)
├── layouts/     # Page layouts (header, footer, pages, posts)
└── main.scss    # Used to import partials and dependencies

  • JavaScript in the theme supports ES6, but is not required (you can write JavaScript as you're used to — we'll go over more about this soon)

  • JavaScript in the theme is setup with DOM-based routing, as originally discussed by Paul Irish

    assets/scripts/
├── customizer.js  # JS specifically for the WordPress theme customizer
├── main.js        # Used to setup the DOM-based routes
├── routes/
│   ├── Common.js  # Used on entire site
│   └── Home.js    # Used on page with 'home' in <body> class
└── util/

  • Always use the wp_enqueue_scripts hook to add new CSS & JavaScript files to your theme, even if they're hosted on a CDN

  • Always keep accessibility in mind
